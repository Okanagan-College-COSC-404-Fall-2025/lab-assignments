Part Two: SQL Review Exercises

The purpose of this part of your assignment is to review SQL and normalization. During the first activity you will implement a database in your schema. During the second and third activities you will write SQL queries to answer several questions.
  1. Consider a flawed database that keeps track of the information about the departments and divisions within an organization. The database has the following schema (# indicates primary key column(s)):
    • L1_DEPARTMENTS(#ID, mgr_ID, div_ID, room, bldg_CODE, dept_campus)
    • L1_DIVISIONS(#ID, div_name, bldg_CODE, bldg_name, div_campus, director_ID)
    • L1_ROOMS(#ROOM_NO, #bldg_CODE, status, area)
    • L1_BUILDINGS(#CODE, bldg_ name, campus)
    • L1_EMPLOYEES(#ID, name, position, dept_ID)
Here are some business rules for the organization (entities are indicated using capitalization):
    • The organization is located at a number of so-called campuses. Each campus is identified by its unique name.
    • Departments are grouped into Divisions. Each Division can oversee many Departments, but each Department is part of only a single Division.
    • Each Department has an office. Although the staff in each Department can occupy many offices, only one office is considered the Department's location. This means that a single Department is located at the campus associated with the Department's office location (i.e. its office's Room location).
    • Each Room is located in a Building.
    • Each room has a ROOM_NO unique to the Building within which it is located.
    • Each building is located on one of the campuses.
    • The Departments that comprise a Division can be located in different buildings (which may be on different campuses). However, each Division must be located in the same building as one of it's departments.
To help you further understand the database, here is some more information about columns of the tables.
  • L1_DEPARTMENTS table: div_ID refers to the ID of a Division. room and bldg_CODE together refer to a specific Room listed in the ROOMS table, dept_campus refers to the campus on which that department's office is located. mgr_ID is the ID of the employee who manages the department.
  • L1_DIVISIONS table: div_campus refers to the campus' unique campus name. director_id refers to the employee ID of the Division's director. bldg_CODE and bldg_name are the CODE and name of the building that is associated with the division's location. div_campus is the campus on which the division is located.
  • L1_ROOMS table: ROOM_NO and bldg_CODE uniquely identify a Room across the organization. ROOM_NO refers to the room number of a room within a building. bldg_CODE refers to CODE in the L1_BUILDINGS table. status refers to one of five single-character values ('O' - occupied office; 'U' - unoccupied office; 'L' - laboratory; 'T' - technical services; 'S' - storage). area refers to the square footage of the room.
  • L1_BUILDINGS table: CODE is a unique 3-letter identifier for a Building. bldg_name refers to a building's name. campus refers to the name of the campus that the Building is on.
